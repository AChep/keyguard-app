Keep files alongside the items they belong to by attaching them directly to vault entries. A recovery code image, a scanned document, or a key file lives next to the login or note it relates to, so everything stays in one place.

When you download an attachment, it is downloaded in the background, non interupting your current workflow.

Additionally you can open an attachment in-app, so it is rendered inside the app rather than written to your device first. This keeps sensitive files off local storage, where other apps or backups might pick them up.

<Screenshot src={previewImage} caption="Previewing an image attachment in-app, without writing it to disk first" />

Images aren't the only thing you can open. Text and Markdown files render inline too, with both raw and formatted views with a basic syntax highlighting, so a recovery note or a config file is readable at a glance.
